# 06 Focus

Focus selects what deserves attention before understanding. It filters
representations into bounded attention surfaces.

## Owns

- Attention boundaries.
- Review queues.
- Candidate grouping and priority surfaces.
- Bounded selection for human or engine attention.

## Does Not Own

- Truth preservation.
- Final interpretation.
- Navigation decisions.
- Actions.

## Current Evidence

- Knowledge Governance review views.
- Candidate Page concept in `canon/ontology.md`.
- First Contact scale lessons showing unbounded candidate intake is not
  reviewable.

## Gaps

- Attention Filter is a concept, not yet a reusable engine/service.
- Candidate Pages remain proposed despite ADR-0005 acceptance.

## Migration Path

Implement Candidate Pages before larger knowledge-review workflows. Treat page
generation as focus, not understanding.
