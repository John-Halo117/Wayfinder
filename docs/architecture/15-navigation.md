# 15 Navigation

Navigation chooses where to go based on missions, bearings, reasoning,
capabilities, constraints, and context.

## Owns

- Route selection.
- Recommendations.
- Navigation posture.
- Tradeoff presentation for route choice.

## Does Not Own

- Capability registry ownership.
- Reasoning.
- Commitments.
- Direct reality mutation.

## Current State

Jarvis is the future navigation engine. Current repository evidence is mostly
contracts, README boundaries, legacy ingress material, and generated Knowledge.

## Health Rule

Views must not perform navigation. Navigation consumes views and reasoning; it
does not live inside presentation.
