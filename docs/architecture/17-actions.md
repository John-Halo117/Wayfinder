# 17 Actions

Actions modify reality. They are downstream of navigation and experiences, and
their effects must be re-observed.

## Owns

- Action requests.
- Execution boundary.
- Side-effect acknowledgement.
- Re-observation expectation.

## Does Not Own

- Direct ARK mutation.
- Reality preservation bypasses.
- Navigation decisions.

## Action Health

No canonical action engine is implemented. Legacy ARK/Foundry contain action
paths such as emitters, scripts, MCP tools, and deployment commands.

## Required Rule

Actions affect reality only. They do not directly mutate ARK. ARK observes and
preserves the results after re-observation.

## Migration Path

Classify legacy action pathways into:

- external adapters
- operations workflows
- Foundry engineering actions
- future Jarvis action recommendations
- policy-gated execution
