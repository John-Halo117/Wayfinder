# 11 Digital Twins

Digital Twins are identity-backed, representation-rich projections of assets,
systems, people, places, or processes.

## Owns

- Twin continuity surfaces.
- State projections.
- Relationship to RID, Asset, Context, Representation, and Provenance.

## Does Not Own

- Asset identity itself.
- Reality preservation.
- Source observation.
- Navigation or action authority.

## Current State

Digital Twins appear in architectural language and generated Knowledge, but no
dedicated implementation owner is active.

## Migration Path

Do not implement a Twin engine first. Build from:

1. RID and Asset model.
2. Storage service.
3. Representation and WEAVE topology.
4. Views and Capsules.
