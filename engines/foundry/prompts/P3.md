# P3

## Mission

Analyze inherited models and knowledge artifacts to discover architectural
improvement opportunities.

## Inheritance

- Prerequisite: execute bootstrap.
- Bootstrap artifacts are authoritative.
- P0 artifacts.
- P1 artifacts.
- P2 artifacts.
- P2.5 artifacts.

## Inputs

- Constitutional Model.
- Repository Model.
- Canonical Knowledge Graph.
- Knowledge Integrity Report.
- Human Decision Queue.
- Repository documentation.
- Existing ADRs.

## Objectives

- Determine constitutional alignment opportunities.
- Identify evidence-backed architectural observations.
- Keep observations separate from implementation work.

## Tasks

- Evaluate constitutional alignment.
- Recover architectural intent.
- Analyze responsibilities.
- Discover shared capability candidates.
- Factor duplicated patterns recursively.
- Check existing capabilities first.
- Record cheapest constitutional approach.
- Identify unprioritized bundle candidates.
- Produce architectural notebook observations.

## Validation

- Opportunities are evidence-backed.
- Observations trace to constitutional principles.
- Existing capabilities are evaluated first.
- No implementation work is performed.
- Observations remain distinct from tasks.

## Outputs

- Architecture Notebook.
- Opportunity Catalog.
- Shared Capability Catalog.
- Recursive Factorization Report.
- Constitutional Alignment Report.
- Layer Violation Report.
- Reuse Analysis.
- Bundle Candidates.
- Architectural Archaeology Report.
- P4 Readiness Report.

## Prohibited

- Generating code.
- Refactoring.
- Modifying runtime behavior.
- Producing pull requests.
- Prioritizing opportunities.
- Estimating schedules.
- Planning migrations.
- Bundling implementation work.

## Success

Can P4 inherit evidence-backed architectural opportunities without any
implementation decisions having been made?
