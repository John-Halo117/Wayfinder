# P2

## Mission

Construct the Canonical Knowledge Graph artifact from inherited bootstrap and
repository discovery artifacts.

## Inheritance

- Prerequisite: execute bootstrap.
- Bootstrap artifacts are authoritative.
- P0 artifacts.
- P1 artifacts.

## Inputs

- Supported participant sources.
- Repository documentation.
- Existing generated `Knowledge/` artifacts.
- Historical artifacts.

## Objectives

- Preserve knowledge sources as evidence.
- Organize derived graph artifacts.
- Record provenance, confidence, lineage, promotion state, conflicts, and
  uncertainty.

## Tasks

- Acquire source inventories.
- Normalize without rewriting history.
- Extract objects and relationships.
- Compile the canonical graph artifact.
- Record conflicts and promotion state.
- Do not rediscover bootstrap or repository knowledge.

## Validation

- Objects have provenance and stable identity.
- Relationships reference valid objects or are recorded as unresolved.
- Historical artifacts remain unchanged.
- Embeddings remain retrieval indexes only.

## Outputs

- Canonical Knowledge Graph.
- Eisengarten Corpus.
- Wayfinder Corpus.
- Historical Corpus.
- Relationship Graph.
- Timeline.
- Promotion Report.
- Conflict Report.
- Knowledge Inventory.
- Extraction Report.

## Prohibited

- Rewriting historical conversations.
- Deleting source material.
- Modifying historical artifacts.
- Promoting implementation details to ontology.
- Merging concepts without evidence.
- Removing provenance.
- Introducing constitutional doctrine.

## Success

Can future phases answer what is known, where it came from, how it evolved, and
how confident Foundry is?
