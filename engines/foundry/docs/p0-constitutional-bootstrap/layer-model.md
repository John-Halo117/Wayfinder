## Evidence Pipeline

The architecture documents refine the stack into an evidence pipeline:

```text
Reality
-> Perception
-> Observations
-> ARK
-> Representations
-> Focus
-> Understanding
-> Missions
-> Bearings
-> Reasoning
-> Navigation
-> Experiences
-> Actions
-> Reality is re-observed
```

Each layer has one primary purpose:

- Perception detects or receives source reality.
- Observations record encounters without interpretation.
- ARK preserves observations, source artifacts, and source relationships.
- Representations produce rebuildable projections.
- Focus selects bounded attention.
- Understanding proposes or reviews meaning.
- Missions bind work to objectives.
- Bearings maintain orientation under uncertainty.
- Reasoning evaluates tradeoffs and claims.
- Navigation recommends routes.
- Experiences present surfaces.
- Actions affect reality and therefore return to observation.

## Engine Execution Grammar

Engines follow:

```text
Ingress -> Reality -> Ephemeral -> Proof -> Promotion -> Core -> Egress
```

- Ingress receives inputs.
- Reality preserves or references source evidence.
- Ephemeral computation may explore or derive.
- Proof validates.
- Promotion makes durable.
- Core contains durable engine responsibility.
- Egress exposes validated outputs.
