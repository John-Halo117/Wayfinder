# Priority Catalog

## Purpose

Catalog bundles by priority after applying dependency, constitutional,
engineering value, mission impact, cost, risk, TRISCA, and TriAvg evidence.

## Required Fields

| Field | Meaning |
| --- | --- |
| Bundle ID | Stable identifier inherited from P4. |
| Priority Tier | Recommended tier for execution sequencing. |
| Dependency Status | Blocking, blocked, independent, parallelizable, or critical-path. |
| Constitutional Justification | Principle or invariant supporting priority. |
| Engineering Value | Reuse, debt reduction, leverage, or simplification value. |
| Mission Impact | Continuity, maneuverability, optionality, capability, or health impact. |
| Relative Cost | Documentation, configuration, reuse, composition, refactor, or new implementation. |
| Risk | Migration, compatibility, regression, complexity, and testing risk. |
| Confidence | Evidence confidence and uncertainty notes. |
