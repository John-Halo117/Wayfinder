════════════════════════════════════════════════════════════════════════════════

This is a production-grade Docker Compose stack with:
  ✓ Traefik v3 (reverse proxy, TLS, routing)
  ✓ Authelia (SSO authentication, 2FA)
  ✓ Navidrome (music streaming server)
  ✓ Home Assistant (event ingestion, automation)
  ✓ Redis (session store)

All traffic encrypted, authenticated, and properly routed through Traefik.
Auto-generated configs, full validation, production-ready.

════════════════════════════════════════════════════════════════════════════════

  - Ubuntu 20.04 LTS or newer (or equivalent Linux)

  - 50GB+ free disk space (for music library + Navidrome cache)

  - Port 80/443 open on firewall (required for Let's Encrypt)

STEP-BY-STEP DEPLOYMENT
════════════════════════════════════════════════════════════════════════════════

STEP 1: CLONE & SETUP REPOSITORY
─────────────────────────────────────────────────────────────────────────────

  # Create deployment directory
  mkdir -p ~/music-stack && cd ~/music-stack

  # Clone or copy all files to ~/music-stack/
  # Files needed:
  #   docker-compose.yml
  #   .env.prod
  #   init-prod.sh
  #   deploy-prod.sh
  #   monitor-prod.sh
  #   backup-prod.sh
  #   traefik/ (directory)
  #   authelia/ (directory)

  # Verify all files present
  ls -la

STEP 2: CONFIGURE ENVIRONMENT (.env)
─────────────────────────────────────────────────────────────────────────────

  # Copy production template
  cp .env.prod .env

  # Edit with your values
  nano .env

  Required values to update:
    DOMAIN=music.example.com              (your domain; must be registered)
    TOP_LEVEL_DOMAIN=example.com          (parent domain for cookies)
    LETSENCRYPT_EMAIL=admin@example.com   (your email; cert expiry notices)
    MUSIC_PATH=/srv/navidrome/music       (music library location)
    DATA_PATH=/srv/navidrome/data         (persistent data directory)
    REDIS_PASSWORD=                       (generate: openssl rand -base64 32)
    AUTHELIA_TOTP_SECRET=                 (generate: openssl rand -base64 32)

  Generate random passwords:
    openssl rand -base64 32
    # Run twice, copy-paste into .env

  Save and exit (Ctrl+O, Enter, Ctrl+X in nano)

STEP 3: SETUP DNS
─────────────────────────────────────────────────────────────────────────────

  # Get your server's public IP
  curl -s https://ifconfig.me

  # Add DNS A record to your domain registrar (e.g., GoDaddy, Namecheap, Route53)
  # Domain: music.example.com
  # Record: A
  # Value: YOUR_SERVER_IP

  # Verify DNS resolves (wait 5-10 minutes for propagation)
  nslookup music.example.com
  # Should return your server IP

STEP 4: VERIFY FIREWALL (Critical!)
─────────────────────────────────────────────────────────────────────────────

  # Open ports 80 and 443 (required for Let's Encrypt and HTTPS)

  If using iptables:
    sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
    sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT

  If using ufw (Ubuntu):
    sudo ufw allow 80/tcp
    sudo ufw allow 443/tcp
    sudo ufw reload

  If using AWS Security Groups / Digital Ocean Firewall:
    - Add Inbound rule: HTTP (80) from 0.0.0.0/0
    - Add Inbound rule: HTTPS (443) from 0.0.0.0/0

  Verify ports open:
    curl http://localhost:80
    curl https://localhost:443

STEP 5: INITIALIZE STACK (Auto-generates all configs)
─────────────────────────────────────────────────────────────────────────────

  # Run initialization script (creates directories, generates configs, validates)

  Expected output:
    ✓ Docker daemon running
    ✓ Directories created
    ✓ acme.json created (mode 600)
    ✓ traefik/traefik.yml generated
    ✓ traefik/dynamic.yml generated

    ✓ All checks pass

    ✓ Services started
    ✓ All services running
    ✓ Endpoints responding

  # password: "$2a$12$..." → password: "YOUR_NEW_HASH"

COMMON ISSUES & FIXES
════════════════════════════════════════════════════════════════════════════════

Issue: "Connection refused" on HTTPS
  Cause: Firewall ports not open
  Fix: Verify ports 80/443 open (netstat -tln | grep -E ':80|:443')

Issue: "Too many failed authorizations" from Let's Encrypt
  Cause: Using production certs before ready
  Fix: Delete /srv/traefik/acme.json, restart with staging

Issue: Authelia "connection refused"
  Cause: Redis not running or password wrong
  Fix: docker-compose logs redis
       Verify REDIS_PASSWORD in .env matches docker-compose.yml

Issue: Navidrome can't read music library
  Cause: File permissions
  Fix: chmod -R 755 /srv/navidrome/music
       chown -R 1000:1000 /srv/navidrome/data

Issue: "502 Bad Gateway" from Traefik
  Cause: Backend service not responding
  Fix: docker-compose ps (check all services up)
       docker-compose logs traefik (check routing)

Issue: Certificates not auto-renewing
  Cause: Port 80 blocked or DNS issues
  Fix: Verify ports open
       Check /srv/traefik/logs/traefik.log for errors

DISASTER RECOVERY
════════════════════════════════════════════════════════════════════════════════

If everything breaks:

  1. Stop containers:
     docker-compose down

  2. Check disk space:
     df -h /srv

  3. View logs:
     cat /srv/traefik/logs/traefik.log
     cat /srv/authelia/logs/authelia.log
     cat /srv/navidrome/logs/navidrome.log

  4. Restart:
     docker-compose up -d

  5. Wait 30 seconds for health checks
     docker-compose ps

  6. Monitor logs:
     docker-compose logs -f


PRODUCTION NOTES

Security:
  - Never expose services without Authelia
  - Always use HTTPS (port 443)
  - Change default passwords immediately
  - Enable 2FA for all users
  - Run backups daily

Performance:
  - Redis memory limit: 256MB (configurable in docker-compose.yml)
  - Navidrome CPU: ~1 core at peak
  - Traefik CPU: <0.5 core
  - Expected RAM: ~1GB total

Scaling:
  - For 100k+ songs: increase Navidrome memory & CPU
  - For high user count: increase Authelia replicas (run multiple containers)
  - For redundancy: use external Redis + multiple Traefik instances (requires Swarm/K8s)
