# Production Stack - FINAL CHECKLIST & QUICK REFERENCE

## DEPLOYMENT QUICK REFERENCE

```bash
# 1. Setup
cp .env.prod .env
nano .env  # Edit: DOMAIN, TOP_LEVEL_DOMAIN, REDIS_PASSWORD, AUTHELIA_TOTP_SECRET

# 2. DNS (add A record to registrar)
# music.example.com → YOUR_SERVER_IP

# 3. Firewall (open ports)
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp

# 4. Deploy (staging first!)
bash init-prod.sh       # Initialize & validate
bash deploy-prod.sh staging   # Staging certs (no rate limits)

# 5. Test
curl https://music.example.com  # Should redirect to login
docker-compose ps               # All services healthy

# 6. Password (CRITICAL!)
docker run --rm authelia/authelia:4.38.5 authelia hash-password
# Copy hash → authelia/users_database.yml → docker-compose restart authelia

# 7. Production
bash deploy-prod.sh production  # Real Let's Encrypt certs

# 8. Monitor
bash monitor-prod.sh            # One-time health check
bash monitor-prod.sh monitor    # Continuous (Ctrl+C to stop)

# 9. Backup
bash backup-prod.sh             # Manual backup
# Or add to crontab: 0 2 * * * cd ~/music-stack && bash backup-prod.sh
```

## TROUBLESHOOTING QUICK REFERENCE

| Issue | Check | Fix |
|-------|-------|-----|
| "Connection refused" | `netstat -tln \| grep :443` | Open firewall ports |
| "Too many authorizations failed" | `grep acme /srv/traefik/logs/traefik.log` | Use staging, then production |
| Authelia not responding | `docker-compose logs authelia` | Check Redis password in .env |
| Navidrome empty library | `ls /srv/navidrome/music/` | Copy music files to path |
| High disk usage | `docker system df` | Prune: `docker system prune` |
| Certificate not renewing | `/srv/traefik/acme.json exists?` | Check port 80, logs |
| 502 Bad Gateway | `docker-compose ps` | Verify all services healthy |

## BACKUP & RESTORE

```bash

bash backup-prod.sh

# Auto-backup daily at 2 AM
(crontab -l 2>/dev/null; echo "0 2 * * * cd ~/music-stack && bash backup-prod.sh") | crontab -

# Restore (from BACKUP_PATH)
docker-compose down
tar -xzf /srv/backups/navidrome_backup_*.tar.gz -C /
docker-compose up -d
```

## DISASTER RECOVERY

```bash
# Full system restore
docker-compose down -v
rm -rf /srv/*
tar -xzf /srv/backups/navidrome_backup_*.tar.gz -C /
docker-compose up -d

# Partial restore (music library only)
rsync -av /backup/music/ /srv/navidrome/music/

# Certificate regeneration
rm /srv/traefik/acme.json
touch /srv/traefik/acme.json && chmod 600 /srv/traefik/acme.json
docker-compose restart traefik
```
