## 📝 ENVIRONMENT VARIABLES (.env)

**REQUIRED (set before deployment):**
```bash
DOMAIN=music.example.com                    # Your domain
TOP_LEVEL_DOMAIN=example.com                # Parent domain
LETSENCRYPT_EMAIL=admin@example.com         # Email
MUSIC_PATH=/srv/navidrome/music             # Music library
DATA_PATH=/srv/navidrome/data               # Persistent data
REDIS_PASSWORD=<random 32 bytes>            # Generate: openssl rand -base64 32
AUTHELIA_TOTP_SECRET=<random 32 bytes>      # Generate: openssl rand -base64 32
```

**OPTIONAL (defaults OK):**
```bash
TZ=UTC                                      # Timezone
TRAEFIK_LOG_LEVEL=info                      # Log level
AUTHELIA_LOG_LEVEL=info                     # Log level
LETSENCRYPT_ENV=production                  # staging or production
```

## ❓ FAQ

**Q: How long does deployment take?**
A: 5 minutes (init) + 5 minutes (deploy) + 5 minutes (waiting for certs) = ~15 minutes

**Q: Can I use this on existing server?**
A: Yes, use separate directories and update port mappings if needed

**Q: How do I backup?**
A: `bash backup-prod.sh` (manual) or add to crontab (daily)

**Q: What if Let's Encrypt rate limit?**
A: Use staging first (`deploy-prod.sh staging`), then production

**Q: How do I restore from backup?**
A: Extract tarball to /srv/, run `docker-compose up -d`

**Q: Can I scale this?**
A: Yes, add more resources or run multiple instances with external Redis

**Q: Is this secure?**
A: Yes, includes TLS, authentication, 2FA, rate limiting, backups
