DEPLOYMENT
═════════════════════════════════════════════════════════════════

1. Start stack (with logs):
   docker compose up -d && docker compose logs -f

2. Wait for services to be healthy:
   docker compose ps
   (All services should show "Up" in ~30-60 seconds)

3. Verify Traefik certificates:
   docker compose logs traefik | grep -i "certificate"

4. Test services:
   curl -v https://music.example.com
   (Should redirect to Authelia login if not authenticated)

5. First login:
   Username: admin
   Password: (from authelia/users_database.yml)
   Enable TOTP (2FA) for security

FAIL-SAFE: COMMON ISSUES & FIXES
═════════════════════════════════════════════════════════════════

ISSUE: "acme.json: permission denied"
FIX: sudo chmod 600 /srv/traefik/acme.json
     sudo chown 65534:65534 /srv/traefik/acme.json

ISSUE: Let's Encrypt "too many failed authorizations"
FIX: Set LETSENCRYPT_ENV=staging in .env
     Delete /srv/traefik/acme.json
     docker compose restart traefik
     Test with staging certs first

ISSUE: Traefik can't find Docker socket
FIX: Verify docker-compose.yml mounts: /var/run/docker.sock:/var/run/docker.sock:ro
     Check daemon running: docker ps

ISSUE: Authelia "connection refused" to Redis
FIX: Verify redis service is running: docker compose ps redis
     Check Redis password matches .env: grep REDIS_PASSWORD .env

ISSUE: Navidrome "Permission denied" on /music
FIX: Mount is read-only (:ro); verify music files are readable
     ls -l /srv/navidrome/music/

ISSUE: "Port 80 already in use"
FIX: Check for other services: sudo lsof -i :80
     Kill/stop conflicting service, restart stack

ISSUE: "DNS does not resolve"
FIX: Add A record to DNS provider
     Wait 5-10 minutes for propagation
     Test: nslookup music.example.com

ISSUE: Home Assistant can't reach Docker daemon
FIX: Home Assistant on host network (network_mode: host)
     Alternatively, use separate bridge network if discovery not needed

MAINTENANCE
═════════════════════════════════════════════════════════════════

Update image versions:
  docker compose pull
  docker compose up -d

View logs:
  docker compose logs -f traefik
  docker compose logs -f authelia
  docker compose logs -f navidrome

Restart service:
  docker compose restart navidrome

Backup data:
  tar -czf backup-navidrome-$(date +%Y%m%d).tar.gz /srv/navidrome/data
  tar -czf backup-authelia-$(date +%Y%m%d).tar.gz /srv/authelia/data

Stop all:
  docker compose down

Stop all and remove volumes (DESTRUCTIVE):
  docker compose down -v
