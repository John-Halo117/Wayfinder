# Build Bible

## Purpose

Owns canonical specifications.

## Owns

Canonical build specifications, promoted implementation requirements, specification continuity, and specification references.

## Does Not Own

Engineering execution, verification workflow, or runtime operation.

## Inputs

Promoted specifications, architectural decisions, proof references, and Foundry outputs.

## Outputs

Canonical specifications and build doctrine.

## Dependencies

Constitution, Contracts, Promotion Registry, Foundry proof outputs.

## Consumers

Foundry, domains, internal applications, and operators.
