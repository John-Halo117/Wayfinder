# Checklists

Checklists make operations repeatable and auditable.

## Purpose

Support active operating tasks without replacing Build Bible source records or
service history.

## Checklist Classes

- startup
- shutdown
- seasonal transition
- emergency response
- post-storm inspection
- pre-maintenance isolation
- post-maintenance recovery
- operator handoff

## Relationships

- Architectural reviews: [Reviews](../../BUILD_BIBLE/governance/reviews/README.md)
- Maintenance procedures: [Reusable Maintenance Procedures](../../BUILD_BIBLE/domains/maintenance/reusable-maintenance-procedures.md)
- Service records: [Service Records](../../BUILD_BIBLE/lifecycle/service-records/README.md)

## Generation Targets

Printed checklists, mobile task lists, handoff packets, service packets.
