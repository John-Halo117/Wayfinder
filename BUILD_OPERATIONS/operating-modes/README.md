# Operating Modes

Operating modes describe property-specific combinations of operational states.

## Purpose

Describe how a property is intentionally run during normal use, idle periods,
storms, outages, maintenance, emergencies, seasonal transitions, and special
events.

## Required Mode Fields

- mode ID
- purpose
- included systems
- expected operational states
- monitoring expectations
- operator actions
- entry criteria
- exit criteria
- fallback mode

## Relationships

- State model: [Operational State Model](../../BUILD_BIBLE/registries/ontologies/operational-state-model.md)
- Reliability: [Reliability](../../BUILD_BIBLE/lifecycle/reliability/README.md)
- Emergency actions: [Emergency Actions](../emergency-actions/README.md)

## Generation Targets

Mode guides, dashboards, alert routing, seasonal operating packets.
