# SOPs

Standard Operating Procedures describe repeatable property-specific actions.

## Purpose

Provide clear operating steps for physical systems without becoming canonical
design source.

## Required SOP Fields

- SOP ID
- affected scope IDs
- operating mode
- prerequisites
- safety constraints
- steps
- expected result
- failure escalation
- records to update

## Relationships

- Build Bible source: [BUILD_BIBLE](../../BUILD_BIBLE/README.md)
- Operational states: [Operational State Model](../../BUILD_BIBLE/registries/ontologies/operational-state-model.md)
- Service records: `BUILD_BIBLE/lifecycle/service-records/`

## Generation Targets

Printed SOPs, mobile checklists, handoff packets, training guides.
