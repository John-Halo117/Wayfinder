# Emergency Actions

Emergency actions describe immediate property-specific responses to unsafe,
damaging, or continuity-threatening events.

## Purpose

Provide fast, explicit actions during high-stress physical events.

## Required Action Fields

- emergency action ID
- trigger
- affected capability
- immediate safety action
- isolation steps
- fallback mode
- recovery handoff
- contacts or roles
- records to update after stabilization

## Relationships

- Recovery procedures: [Recovery Procedures](../../BUILD_BIBLE/lifecycle/reliability/recovery-procedures.md)
- Operational states: [Operational State Model](../../BUILD_BIBLE/registries/ontologies/operational-state-model.md)
- Dependency ontology: [Dependency Ontology](../../BUILD_BIBLE/registries/ontologies/dependency-ontology.md)

## Generation Targets

Emergency binders, laminated shutoff guides, alert response flows, recovery
handoffs.
