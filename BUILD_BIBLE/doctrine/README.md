# Doctrine

Doctrine contains long-lived design principles for the physical world.

Doctrine is intentionally slow-moving. It explains how to think about physical
systems before specific designs are written.

## Purpose

Define invariant physical design principles that outlive any single property,
building, material, or technology generation.

## Relationships

Doctrine informs contracts, schemas, domain patterns, lifecycle records, and
generated artifact validation.

## Lifecycle

Doctrine changes only through deliberate decision records because it affects
every downstream physical pattern.

## Generation Targets

Doctrine is consumed by validators, review checklists, design guides, and
generated acceptance criteria.
