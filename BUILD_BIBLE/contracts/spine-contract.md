# Spine Contract

A spine is a physical scope's organized service backbone.

## Required Fields

- owning scope ID
- spine ID
- distributed capabilities
- route or location
- access points
- isolation points
- monitoring points
- reserve capacity
- known bottlenecks
- service procedure links
- last verified reality

## Invariants

- A spine must be inspectable along documented access points.
- Hidden runs must have known endpoints and route evidence.
- Spare paths must be labeled as usable, blocked, or unknown.
