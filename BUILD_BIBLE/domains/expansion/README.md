# Expansion Domain

Owns future expansion reserves, protected corridors, stub-outs, uncommitted
capacity, and blocked-option records.

Canonical here:

- expansion zones
- reserved routes
- future interfaces
- protected structural allowances
- capacity held for unknown future use
- decisions that spend optionality

## Purpose

Define how new physical capabilities are added while consuming reserved
interfaces before reconstruction.
