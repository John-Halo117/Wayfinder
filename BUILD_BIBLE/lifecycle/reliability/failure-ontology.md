# Failure Ontology

Failure is the loss, degradation, or unsafe behavior of a capability outside
accepted operating limits.

## Failure Classes

- sudden failure
- gradual degradation
- intermittent failure
- capacity exhaustion
- control failure
- monitoring failure
- access failure
- dependency failure
- environmental failure
- human-operation failure

## Required Failure Questions

- What failed?
- What capability was lost?
- How was it detected?
- How is it isolated?
- How is it repaired?
- How is service recovered?
- What degraded before failure?
- What downstream systems are affected?
- What documentation or digital twin state changes?
