# Repository Validation

Repository validation defines reusable checks for Build Bible source quality.

P3 documents the validation model. It does not implement validators.

## Contents

- [Repository Validation Standard](repository-validation-standard.md)
- [Architectural Linter Rules](architectural-linter-rules.md)

## Relationships

- Intelligence: [Repository Intelligence](../intelligence/README.md)
- Quality reports: [Quality Reports](../reports/README.md)
- Traceability: [Traceability Standard](../reviews/traceability-standard.md)

## Lifecycle

Validation rules evolve through Engineering Decision Records when they change
what counts as a valid Build Bible source record.

## Generation Targets

Validation reports, lint reports, quality dashboards, readiness reports, and
compiler preflight checks.
