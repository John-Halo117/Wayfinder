# Document Metadata Standard

Every Build Bible document is a graph node.

## Required Metadata

Each major canonical document should expose or imply:

- document ID
- title
- layer
- canonical owner
- purpose
- parent documents
- child documents
- implemented contracts
- referenced schemas
- related patterns
- generated artifact targets
- verification expectations
- review expectations
- lifecycle status

## Layer Values

- doctrine
- contract
- schema
- ontology
- pattern
- standard
- specification
- reality record
- lifecycle record
- operations reference
- generated artifact
- template
- index
- report

## Rule

Metadata is for discovery and validation. It must not duplicate canonical
technical facts owned elsewhere.
