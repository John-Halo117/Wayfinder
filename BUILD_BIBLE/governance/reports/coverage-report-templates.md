# Coverage Report Templates

Coverage reports show where the Build Bible is complete enough for review,
validation, or generation.

## Coverage Report Types

- doctrine coverage
- contract coverage
- schema coverage
- pattern coverage
- verification coverage
- reliability coverage
- digital twin readiness
- generated artifact readiness
- traceability coverage
- relationship coverage

## Required Report Fields

- report ID
- report type
- evaluated scope
- source documents
- coverage criteria
- coverage result
- gaps
- blockers
- recommended next records
- generated date

## Result States

- `complete`
- `partial`
- `missing`
- `blocked`
- `not-applicable`

## Relationships

- Validation: [Repository Validation Standard](../validation/repository-validation-standard.md)
- Digital twin readiness: [Digital Twin Readiness Standard](../../digital-twin/readiness-standard.md)
- Compiler model: [Compiler Model](../intelligence/compiler-model.md)
