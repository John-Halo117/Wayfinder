# Verification

The Build Bible preserves reality by separating intent from verified condition.

## Verification States

- `unverified`: stated but not checked against physical evidence.
- `observed`: seen or measured, with evidence attached.
- `inferred`: derived from evidence but not directly observed.
- `designed`: intended future state.
- `installed`: implemented and observed after work.
- `retired`: no longer active, with retirement history preserved.

## Evidence Requirement

Any claim about existing physical condition must link to an evidence record in
`reality/` or state that it is unverified.

## Last Verified Reality

Every active physical scope must eventually carry:

- date
- verifier
- evidence link
- verification state
- known uncertainty
