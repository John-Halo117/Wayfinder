# Governance

Governance defines how the Build Bible remains coherent over decades of edits,
construction, service, and generated output.

## Owns

- Scope boundaries for physical reality.
- Canonical ownership rules.
- Identity and spatial addressing rules.
- Change-control rules.
- Verification states.
- Generated-artifact boundaries.
- Stable-principle promotion.
- Naming and addressing standards.
- Review tools and evaluation rubrics.
- Repository intelligence.
- Repository validation.
- Quality report templates.

## Does Not Own

- Domain-specific physical designs.
- Generated CAD, BIM, diagrams, or configs.
- Wayfinder software architecture.
