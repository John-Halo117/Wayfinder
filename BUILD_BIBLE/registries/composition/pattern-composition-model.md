# Pattern Composition Model

Composition combines reusable patterns and standards into a higher-level
pattern without creating a deep inheritance chain.

## Composition Fields

- composed pattern
- component patterns
- component standards
- required contracts
- resource flows
- dependencies
- reliability records
- verification standards
- generated artifact targets

## Rule

Composition is explicit. A document that relies on another pattern as a
component should use the `composes` relationship from the
[Knowledge Graph Relationship Model](../../governance/intelligence/knowledge-graph-relationship-model.md).

## Inheritance Versus Composition

- Inheritance: this pattern is a specialized form of another pattern.
- Composition: this pattern is assembled from several patterns or standards.
